# CareerPilot APAC 新手使用手册

这份手册面向第一次使用 CareerPilot APAC 的用户。你不需要先理解代码结构，只要按顺序完成启动、录入资料、导入岗位、准备投递包和使用 Edge 扩展即可。

## 1. 这套工具能帮你做什么

CareerPilot APAC 是一个本地运行的个人求职工作台，重点解决三件事：

- 把简历、教育经历、项目经历、联系方式等重复填写信息集中维护。
- 把中国大陆、新加坡、香港、澳洲和远程岗位放进同一个岗位库，按市场、匹配分和状态管理。
- 在投递前准备材料包，并用 Microsoft Edge 扩展辅助填写安全字段，最后提交仍由你人工确认。

它不是自动代投系统，不会点击最终提交，不会绕过验证码，也不会自动上传简历文件。

## 2. 第一次启动

在项目根目录打开 PowerShell：

```powershell
cd "C:\Users\HECATE\Documents\求职推荐系统 it方向 开发"
npm install
npm run db:generate
npm run db:push
npm run dev
```

看到本地服务启动后，在浏览器打开：

```text
http://localhost:3000
```

如果端口被占用，终端会提示新的端口，按提示打开对应地址。

## 3. 第一次使用顺序

建议第一次按下面顺序操作，不要一开始就直接投递：

1. 打开 `填表资料库`，先维护姓名、邮箱、电话、学校、专业、学历、签证/工作权利、项目经历和常用回答。
2. 打开 `数据源`，用 CSV、表格文本、邮件提醒内容或岗位描述批量导入岗位。
3. 打开 `市场雷达`，检查岗位是否识别出公司、市场、地点、链接、关键词和匹配理由。
4. 打开 `今日打卡`，生成当天投递清单，优先处理匹配分高、截止时间近、签证风险低的岗位。
5. 打开 `投递包`，选择一个岗位，生成材料草稿并确认投递状态。
6. 在 Edge 里打开官方申请页面，用扩展扫描表单、保存快照、填写安全字段。
7. 回到 `投递管线`，把岗位状态更新为已投递、笔试、面试、拒信或 offer。
8. 定期打开 `本地备份`，导出 JSON 备份。

## 4. 导入岗位数据

最稳的导入方式是 CSV 或表格文本。字段可以包含：

```text
市场,公司,岗位,地点,链接,描述,来源,匹配分,发布时间
```

也可以直接粘贴类似下面的文本：

```text
Shopee | Software Engineer Intern | Singapore | https://example.com/job
要求 JavaScript、React、API、数据库，2027 届可投。
```

导入后系统会自动去重。如果同一个链接重复出现，会更新 `lastSeenAt`，不会创建重复岗位。

## 5. 每天 15 分钟使用流程

建议每天只做一轮，不要把时间耗在无限搜索上：

1. 在 `数据源` 刷新或粘贴新岗位。
2. 在 `今日打卡` 生成当天投递任务。
3. 选 3 到 8 个岗位进入 `投递包`。
4. 生成材料草稿，复制需要的 Cover Letter、筛选题回答和 JD 对齐卖点。
5. 用 Edge 扩展辅助填表，敏感字段和最终提交人工确认。
6. 在 `投递管线` 更新状态，并记录下一次跟进时间。

## 6. Edge 扩展安装

CareerPilot APAC 的扩展目录在：

```text
apps/edge-extension
```

安装步骤：

1. 打开 Microsoft Edge。
2. 进入 `edge://extensions`。
3. 打开 `开发人员模式`。
4. 点击 `加载解压缩的扩展`。
5. 选择项目里的 `apps/edge-extension` 文件夹。
6. 回到申请页面，点击浏览器工具栏里的 CareerPilot 扩展按钮。

推荐实战流程：

1. 在 Web 工作台的 `投递包` 页面点击 `设为当前 Edge 申请`。
2. 在 Edge 打开官方申请链接。
3. 点击扩展里的 `保存表单快照`。
4. 点击 `填写安全字段`。
5. 对签证、工作权利、薪资、敏感声明、文件上传和最终提交按钮进行人工确认。

## 7. 投递包怎么用

`投递包` 是单个岗位的操作页。它会集中展示：

- 岗位 JD、公司、地点、市场、链接和匹配分。
- 关键词、签证风险、加分理由和扣分理由。
- 当前申请状态、备注、投递时间和回信时间。
- 推荐简历版本、Cover Letter 草稿、筛选题回答草稿。
- Edge 表单快照、可安全填写字段数、待人工确认字段数。

实际投递时建议只在一个页面里完成这个岗位，不要在多个页面之间反复跳转。

## 8. 材料库和题库

`材料库` 保存不同岗位生成的材料草稿。当前版本先用本地模板生成，不依赖外部 AI API。

`笔试面试库` 适合保存：

- 常见行为面试题。
- 公司笔试题。
- OA 复盘。
- 项目经历 STAR 模板。
- 筛选题常用回答。

投递包生成材料时会优先复用这些本地答案。

## 9. 本地备份

打开 `本地备份` 页面可以导出 JSON 快照。建议在这些场景备份：

- 大批量导入岗位前。
- 修改大量个人资料前。
- 每周固定备份一次。
- 准备重装电脑或迁移项目时。

恢复是合并模式：同 ID 数据会更新，新 ID 数据会插入，不会主动删除你本地已有记录。

## 10. 常见问题

### 页面打不开

确认终端还在运行：

```powershell
npm run dev
```

然后重新打开 `http://localhost:3000`。

### 数据没有保存

先检查是否运行过数据库初始化：

```powershell
npm run db:generate
npm run db:push
```

然后重启开发服务。

### Edge 扩展连不上本地系统

确认 Web 服务正在运行，并且扩展默认连接的是：

```text
http://localhost:3000
```

如果页面是公司内嵌 iframe 或特殊 ATS，先保存表单快照，再回到 Web 工作台查看字段识别结果。

### 中文显示乱码

运行编码健康检查：

```powershell
npm run verify:static
```

如果只是浏览器缓存问题，刷新页面或重启开发服务。

### 想确认系统是否正常

运行完整验证：

```powershell
npm run verify
```

如果只想验证 Edge 扩展：

```powershell
npm run verify:edge
```

## 11. 使用边界

为了避免账号风险和平台条款问题，当前版本固定遵守这些边界：

- 不自动点击最终提交。
- 不绕过验证码、登录墙或访问限制。
- 不自动上传文件。
- 不伪造经历、学历、签证或工作权利信息。
- 不抓取需要登录授权或禁止自动化访问的数据。

这套系统的目标是减少重复劳动，而不是替你做不可控的无人值守投递。
