# CareerPilot APAC 本地使用手册

这份手册面向个人本地使用，不包含商业化后台、多用户、云同步或自动无人投递。所有数据默认保存在本机 SQLite 数据库里，Edge 扩展只连接本地 Web API。

## 启动

在项目根目录运行：

```powershell
npm install
npm run db:generate
npm run db:push
npm run dev
```

Web 工作台地址：

```text
http://localhost:3000
```

首次打开后先看顶部的 `本地启用清单`。它会检查岗位池、填表资料、今日计划、投递包、Edge 表单快照和本地备份是否就绪，并提供下一步跳转。需要归档当前状态时可点击 `复制启用清单`。

`本地运行状态` 会固定显示在工作台顶部，集中展示本地日期、数据库同步、今日计划、Edge 填表资产和可备份记录。点击 `复制运行状态` 可把当天本地状态导出成 Markdown，适合放入日记或周复盘，不会触发任何外部投递动作。

## 全局搜索

工作台顶部的 `全局搜索` 会在本地数据中同时搜索岗位、投递记录、材料草稿、简历版本、题库答案和数据源。可以输入公司、岗位、市场、材料标题、题库关键词或数据源名称。

键盘使用：按 `Ctrl+K` 或 `Cmd+K` 可聚焦搜索框，输入关键词后用方向键选择结果，按 `Enter` 打开当前结果，按 `Escape` 会清空并退出搜索。

搜索结果会直接跳转：

- 岗位、投递和材料结果会打开对应 `投递包`。
- 简历结果会进入 `材料库`。
- 题库结果会进入 `笔试面试库`。
- 数据源结果会进入 `数据源`。

这个搜索只读取本地 SQLite 数据，不会联网搜索岗位，也不会触发自动投递或自动提交。

## 每日流程

1. 打开 `今日打卡`。
2. 先看 `每日岗位刷新`，点击 `刷新公开来源` 更新公司官网、ATS 或邮件提醒公开 URL。
3. 点击 `生成打卡计划`，生成当天短名单。
4. 先看 `今日行动队列`，它会把岗位刷新、到期跟进、短名单和高匹配岗位压成优先动作。
5. 使用 `复制行动队列` 归档今天最该先做的 1-3 件事。
6. 对每个岗位执行 `准备材料`、`打开投递包`、`加入管线`、`标记投递` 或 `跳过`。
7. 使用 `复制今日计划` 或 `下载今日计划.md` 把当天短名单归档。
8. 如果有到期跟进，先处理 `今日跟进` 里的回信、笔试、面试或材料动作。

今日计划 Markdown 会包含公开来源刷新状态、进度、短名单、跟进事项和操作边界，适合放到本地笔记或复盘文档。

## 数据源与岗位导入

`数据源` 页面支持三类本地导入：

- 手动维护数据源，例如公司官网、邮件提醒、ATS 平台导出。
- 剪贴板/邮件导入，适合复制官网岗位、邮件提醒或平台 job alert。
- CSV 文件导入，适合平台导出的表格。
- 公开来源同步，适合你自己配置的公司官网、ATS 公开列表或邮件提醒链接。

CSV 推荐表头：

```text
市场,公司,岗位,地点,链接,描述,匹配分,发布时间
```

导入后页面会显示：

- 总数
- 新增
- 去重
- 跳过

岗位会按市场、公司、岗位和链接生成去重键。重复导入同一岗位时会更新 `lastSeenAt`，不会反复创建重复岗位。

公开来源同步的边界：

- 只访问你在数据源里保存的 `http/https` 公开页面。
- 不使用登录态，不绕过平台限制，不处理验证码。
- 不自动投递，只把页面里识别到的岗位文本导入本地岗位库。
- 同步结果会记录新增、去重、跳过、失败和最近同步日志。
- `今日打卡` 里的 `每日岗位刷新` 使用同一套同步逻辑，适合每天生成计划前先跑一次。

## 市场雷达

`市场雷达` 用来检查和修正入库岗位：

- 搜索公司、岗位、地点和关键词。
- 按市场筛选中国大陆、新加坡、香港、澳洲、远程/全球。
- 修改匹配分、签证风险、毕业生匹配和 JD 描述。
- 对不再处理的岗位勾选归档，避免进入每日短名单。
- `岗位数据健康` 会统计缺链接、JD 太短、低匹配、高风险、长期未见和疑似重复岗位。
- 点击健康卡可以快速筛选对应问题，`复制质量清单` 可把待修岗位整理成 Markdown。
- `导出岗位.csv` 会按当前搜索、市场、归档和质量筛选导出岗位列表，适合放进 Excel/表格做市场对比。

## 投递包

`投递包` 是单个岗位的操作页。它把这些信息放在同一个界面：

- JD、市场、公司、地点、链接、匹配分。
- 解析关键词、加分原因、扣分/风险。
- 申请记录和跟进日期。
- 推荐简历版本和材料草稿。
- Edge 表单快照和填表运行。
- 题库引用。
- 提交前检查清单。
- 绑定快照回放，按“可直接填写 / 待人工确认 / 缺映射或缺值”拆分真实表单字段。

常用动作：

- `生成/刷新材料包`
- `设为当前 Edge 申请`
- `绑定表单快照`
- `复制投递包摘要`
- `下载摘要.md`
- `复制清单`
- `复制回放清单`
- `下载回放.md`
- `人工提交后标记已投递`

投递包 Markdown 适合按岗位归档，便于后续复盘“为什么投、用了什么材料、什么时候跟进”。
回放清单更适合提交前逐项核对：它只解释 Edge 对当前表单的识别和映射状态，不会自动提交申请。

## Edge 扩展

安装方式：

1. 打开 `edge://extensions`。
2. 开启开发人员模式。
3. 选择 `加载解压缩的扩展`。
4. 选择 `apps/edge-extension` 文件夹。

推荐真实申请流程：

1. 在 Web 工作台打开目标岗位的 `投递包`。
2. 点击 `设为当前 Edge 申请`。
3. 在 Edge 打开官方申请页。
4. 打开 CareerPilot 扩展。
5. 先 `扫描字段` 或 `保存表单快照`。
6. 再使用 `填写安全字段`。
7. 敏感字段、文件上传、验证码、最终提交都人工处理。

边界固定：

- 不自动提交申请。
- 不绕过验证码。
- 不自动上传文件。
- 敏感字段只提示人工确认。

## 填表资料库

`填表资料库` 是 Edge 扩展读取的本地资料源。建议在真实申请前先检查 `资料库完整度`：

- `补齐常见字段` 会加入 APAC 求职表单常见字段框架，包括中英文名、微信、地址、作品链接、教育背景、薪资/到岗、渠道、推荐人和合规确认项。
- `复制缺口清单` 会把待补字段整理成文本，适合逐项补真实值。
- `填表速查包` 可按市场导出安全字段、待确认字段和敏感字段提醒，适合 Edge 无法处理或官网字段需要人工粘贴时使用。
- `下载速查.md` 会生成一份 Markdown，本地归档或边填表边查看都可以。
- `当前快照回放` 会把保存过的真实申请表拆成可直接填写、待人工确认、缺映射/缺值三类，并支持复制或下载回放清单。
- 安全字段可以被 Edge 辅助填写，例如姓名、邮箱、电话、学校、作品链接。
- 薪资、到岗、推荐人、工作权利和签证问题默认需要人工确认。
- 国籍、出生日期、证件号码等敏感字段只用于识别提醒，不会进入自动填表资料接口。

补完字段后点击 `保存资料库`。如果在真实申请页遇到识别不准的字段，先用 Edge 保存表单快照，再回到 `字段映射学习` 把该字段映射到正确资料键。学习队列会统计当前快照中可学习、已保存和缺少资料值的字段；对已经识别出资料键且属于安全字段的项目，可以点击 `批量保存安全建议`，后续同类表单会优先使用这些规则。

## 材料库与题库

`材料库` 维护简历版本和岗位材料草稿：

- 简历版本可以按市场和方向区分。
- 材料包当前使用本地模板生成，不依赖外部 AI API。
- 每个岗位可生成 JD 对齐卖点、Cover Letter 草稿和筛选题回答草稿。

`笔试面试库` 用来沉淀可复用答案：

- OA / 技术面 / 行为面 / 签证和工作权利问题。
- 每条答案有安全等级：安全可复用、使用前确认、敏感勿外发。
- 投递包会引用相关题库内容。

## 投递管线与跟进

`投递管线` 记录岗位状态：

- 已收藏
- 已准备
- 已投递
- 笔试
- 面试
- Offer
- 未通过
- 跳过

建议每条申请都维护：

- 投递日期
- 回信日期
- 下一步动作
- 下次跟进日期
- 备注/复盘

有到期或逾期动作时，会显示在 `今日打卡` 的 `今日跟进`。

`投递复盘` 会汇总有效回应率、面试推进率、活跃跟进、超过 14 天未更新的卡住记录，并按市场展示记录数、已投递数、回应数和回应率。点击 `复制投递复盘` 可导出 Markdown，适合每周复盘哪些市场值得继续加码、哪些岗位需要淘汰或补跟进。

`7日跟进节奏` 会把活跃投递按未来一周分组，并额外标出逾期和未排期记录。建议每天打开 `投递管线` 时先看这个面板，再决定先查回信、准备 OA、约模拟面试或补跟进日期。

可用动作：

- `一键排期未排期`
- `复制7日节奏`
- `下载7日节奏.md`
- `导出管线.csv`

`一键排期未排期` 只会处理已准备、已投递、笔试和面试这类活跃记录，把没有跟进日期的记录分配到未来一周，并补一个默认下一步动作。它不会改已排期记录，也不会触发任何外部平台操作。

导出的 Markdown 和 CSV 只用于本地提醒、表格分析和复盘，不会触发自动投递、自动提交或绕过验证码。

## 本地备份

`本地备份` 页面支持：

- 自动检查中文编码和数据库异常。
- 查看本地维护命令。
- 导出当前 SQLite 数据为 CareerPilot APAC JSON。
- 恢复前预检备份结构。
- 合并恢复备份。

恢复是 merge-only：

- 同 ID 记录会更新。
- 新 ID 记录会新增。
- 不会清空或删除当前本地数据。

建议在大量导入岗位、修改资料库或重装系统前导出备份。

`本地健康检查` 面板会显示：

- 中文编码健康。
- 当前可备份记录数量。
- 完整验收命令。
- smoke 测试数据清理边界。

面板里的 `复制维护命令` 会复制常用命令，适合在调试或阶段验收前直接运行。

## 验证与清理

完整验收：

```powershell
npm run verify
```

单独检查：

```powershell
npm run typecheck
npm run lint
npm run build
npm run api:smoke -w @careerpilot/web
npm run visual:smoke -w @careerpilot/web
npm run smoke:cleanup -w @careerpilot/web
```

`api:smoke` 和 `visual:smoke` 会真实测试 CSV 导入、备份导出、备份预检、合并恢复、页面下载、ATS Lab 和本地 API。ATS Lab 当前覆盖 Workday、Greenhouse、Lever、SmartRecruiters、Ashby、BambooHR 和通用公司官网表单。测试数据使用 `__careerpilot_smoke_*` 命名，并在测试结束后自动清理。

如果中断测试或怀疑本地库里残留测试数据，可以手动运行：

```powershell
npm run smoke:cleanup -w @careerpilot/web
```

验收报告位置：

```text
output/verification/verify-report.json
output/verification/ACCEPTANCE.md
```

## 当前边界

当前版本是个人本地 MVP：

- 不做登录、多用户、支付或商业化后台。
- 不接付费岗位数据库。
- 不违规爬取平台。
- 不自动提交申请。
- 不绕过验证码或平台限制。
- 不自动上传简历文件。

岗位数据优先来自你可合法访问和导出的来源：CSV、剪贴板、邮件提醒、公司官网公开页面和平台导出内容。
